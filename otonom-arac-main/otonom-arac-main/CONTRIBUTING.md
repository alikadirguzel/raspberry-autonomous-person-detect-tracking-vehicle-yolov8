# Katkıda Bulunma Rehberi

Bu projeye katkıda bulunmak istediğiniz için teşekkürler!

## Nasıl Katkıda Bulunabilirim?

1. **Issue Açın**: Yeni bir özellik veya bug için issue açın
2. **Fork Edin**: Repository'yi fork edin
3. **Branch Oluşturun**: Yeni bir feature branch oluşturun
4. **Değişiklik Yapın**: Kodunuzu yazın ve test edin
5. **Commit Edin**: Anlamlı commit mesajları yazın
6. **Push Edin**: Değişikliklerinizi push edin
7. **Pull Request Açın**: PR açın ve değişikliklerinizi açıklayın

## Kod Standartları

- Python PEP 8 standartlarına uyun
- Fonksiyon ve sınıf isimleri açıklayıcı olsun
- Yorumlar Türkçe veya İngilizce olabilir
- Her yeni özellik için test ekleyin (mümkünse)

## Commit Mesajları

- Kısa ve açıklayıcı olun
- Örnek: "feat: Kalman filtresi iyileştirmeleri"
- Örnek: "fix: Motor kontrol hız limiti düzeltmesi"
- Örnek: "docs: README güncellemesi"

## Pull Request Süreci

1. PR açmadan önce kodunuzun çalıştığından emin olun
2. PR açıklamasında ne değiştiğini detaylı anlatın
3. İlgili issue'ları referans edin
4. Kod review için hazır olun

## Sorularınız mı var?

Herhangi bir sorunuz varsa, lütfen bir issue açın veya mevcut issue'lara yorum yapın.
